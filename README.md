# GALITSKIY & GALITSKIY — сайт премиальной винодельни

> Имиджевый SPA с высокой плотностью анимации для премиального бренда. Кастомный scroll-controller, динамический звук, Lottie-анимации, SSR на Express + интеграция Bitrix API. Lighthouse 90+.

**Продакшн:** [galitskiy-galitskiy.ru](https://galitskiy-galitskiy.ru)

---

## О проекте

Имиджевый сайт для премиальной винодельни GALITSKIY & GALITSKIY. Проект с повышенными требованиями к анимациям, производительности и визуальному качеству — каждый элемент интерфейса усиливает ощущение премиального продукта.

Основная инженерная задача: реализовать плотную анимационную систему с динамическим звуком и 3D-элементами без деградации производительности. Lighthouse 90+ при высокой визуальной насыщенности.

## Архитектура

```
src/
├── components/         # Vue-компоненты
├── controllers/        # Кастомный scroll-controller, анимационная система
├── sections/           # Секции страницы
├── styles/             # Глобальные SCSS + модули по компонентам
├── api/                # Интеграция с Bitrix API
└── server/             # Express SSR + роутинг
gulp.dev.js             # Конфигурация сборки (Gulp + esbuild)
```

**Ключевые решения:**

- **Кастомный scroll-controller** — написан с нуля, управляет последовательностью анимаций по скроллу без тяжёлых библиотек типа GSAP ScrollTrigger. Даёт точный контроль над тайминг-кривыми и очерёдностью элементов.
- **Единая анимационная система** — scroll-анимации, Lottie и переходы управляются через единый контроллер. Нет разрозненных setTimeout и addEventListener по всей кодовой базе.
- **Динамический звук через Howler.js** — звуковые эффекты привязаны к пользовательским действиям и анимационным триггерам, загружаются лениво.
- **SSR на Express** — страницы рендерятся на сервере, что ускоряет первый рендер и обеспечивает SEO-индексацию.
- **Gulp + esbuild** — кастомный билд-пайплайн вместо стандартного Vite. Полный контроль над обработкой ассетов и шагами оптимизации.

## Анимационная система

Сайт построен вокруг трёх слоёв анимации, работающих синхронно:

**Scroll-анимации** — кастомный контроллер отслеживает позицию скролла и управляет состоянием каждого элемента. Плавные входы, параллакс, pinned-секции.

**Lottie** — сложные векторные анимации рендерятся через lottie-web. Синхронизированы с позицией скролла через контроллер.

**3D через Spline** — интерактивные 3D-объекты встроены в секции без деградации FPS благодаря ленивой загрузке и контролю видимости.

## Интеграция с Bitrix API

- Динамическая загрузка контента через Bitrix REST API
- SSR-рендеринг данных — контент доступен краулерам без JS
- Обработка ошибок и fallback на статический контент

## Локальный запуск

```bash
# Установка зависимостей
npm install

# Настройка окружения
cp .env.example .env
# Заполнить: BITRIX_API_URL, BITRIX_API_KEY

# Dev-сервер с hot reload
npm run dev

# Продакшн-сборка
npm run build

# Запуск сервера
npm run server
```
